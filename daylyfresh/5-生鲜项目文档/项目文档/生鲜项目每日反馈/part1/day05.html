项目day05反馈

  • 1.理解网站性能优化的两种方案。

    网站性能优化的两种方案:
  1) 页面静态化: 查询数据库中的数据渲染出页面内容之后，将页面内容保存成一个静态文件。
  2）缓存: 从数据库中查询出数据之后，把数据放在缓存中，再次使用的时候直接从缓存中获取数据。

网站性能优化的本质:
  减少数据库的查询。

页面静态化相关知识:
  1）使用celery生成静态首页。
  2）配置Nginx提供静态首页。
    location / {
        # root指定网站的根目录
        root '静态首页存放的目录';
        # index指定如果直接通过ip:port访问网站，默认返回root目录下的index.html
        index index.html;
    }

    location /static {
        # 指定静态文件存放的目录
        alias '静态文件存放的目录';
    }
  3）静态首页重新生成。
    当管理员通过admin管理页面修改了首页用到的数据表的数据时，需要重新生成静态首页。
    提示: 自定义对应的模板admin管理类，重写ModelAdmin中的save_model和delete_model。
  4）静态首页和IndexView的区分。
    通过配置调度的Nginx服务器，把静态首页和IndexView区分开。

缓存相关知识:
  1）Django中设置缓存。
    配置CACHES指定缓存数据存储在哪里(内存型数据库，关系型数据库，文件，本机内存)。
  2）Django缓存级别。
    2.1 站点级缓存
    2.2 单个view缓存
    2.3 模板片段缓存
  3）Django中操作缓存的api。
    from django.core.cache import cache
    设置缓存: set(缓存名称，缓存数据，有效时间)
    获取缓存: get(缓存名称)
    清除缓存: delete(缓存名称)

  • 3.理解商品列表页面中列表页页码的控制。

    前端传递的参数：种类id(type_id) 页码(page) 排序方式(sort)
商品列表页的url地址: '/list/种类id/页码?sort=排序方式'

列表页注意点:
  1）列表页页码控制:  如果分页之后页码过多，最多页面上显示5个页码(当前页前2页，当前页，当前页后2页)。
  2）页码列表处理4种情况:
    2.1 分页之后总页数不足5页，显示所有页码。
    2.2 如果当前页是前3页，显示1-5页。
    2.3 如果当前页是后3页，显示后5页。
    2.4 其他情况，显示当前页前2页，当前页，当前页后2页。
    num_pages = paginator.num_pages
    if num_pages < 5:
        pages = range(1, num_pages+1)
    elif page <= 3:
        pages = range(1, 6)
    elif num_pages - page <= 2:
        pages = range(num_pages-4, num_pages+1)
    else:
        pages = range(page-2, page+3)
