每日反馈内容

day01反馈

  • 1.建立web网站开发中请求【(客户端)-应答(服务器)，http协议(请求报文和应答报文)，web服务器和web框架的概念(wsgi协议)】的概念
  • 2.了解web MVC框架中MVC各部分的功能【M:模型，V:视图，C:控制器】
  • 3.掌握Django 框架中MVT各部分的功能【M:模型，V:视图，T:模板】
    M(Model)：模型，和数据库进行交互。
V(View)：视图，接收请求，进行处理，和M和T进行交互，返回应答。
T(Template)：模板，产生html页面。
  • 4.了解虚拟环境的搭建并掌握其使用【mkvirtualenv, deactivate, workon, rmvirtualenv】
    概念：虚拟环境是真实python环境的复本。
作用：在虚拟环境中安装python包，安装到了虚拟环境的python中。
  • 5.掌握Django项目的创建【项目创建，应用创建，应用注册】
    创建项目：django-admin startproject 项目名称
创建应用：python manage.py startapp 应用名称
注册应用：修改项目的settings.py配置文件
INSTALLED_APPS = (
  # ...
  '应用名称', # 注册应用
)
启动开发web服务器：python manage.py runserver

  • 6.掌握视图view的使用【视图函数定义，url配置，url匹配的过程】

    视图函数定义:
  def 视图函数名(request):
    # 进行处理...
    return HttpResponse('返回给浏览器显示的内容')

url配置:
1) 项目的urls.py中包含应用的urls文件。
urlpatterns = [
  # ...
  url(r'^url地址正则表达式', include('应用用的urls文件')),
  # ...
]
2）应用的urls文件中写url地址和视图函数的对应关系。
urlpatterns = [
  # ...
  url(r'^url地址正则表达式', '视图函数名'),
  # ...
]

  • 7.掌握模板template的使用【模板目录配置，模板使用过程，render简写函数，模板变量和模板for循环】

      模板目录配置:
  TEMPLATES = [
    {
        # ...
        'DIRS': [os.path.join(BASE_DIR, 'templates')],
        # ...
    }
  ]

  模板使用过程:
  1) 加载模板文件，获取模板对象
  from django.template import loader
  temp = loader.get_template('模板文件路径')
  2）定义模板上下文，给模板文件传递数据
  from django.template import RequestContext
  context = RequestContext(request, {})
  3) 模板渲染：替换模板变量，产生替换后的html页面
  res_html = temp.render(context)

  render简写函数:
  from django.shortcuts import render
  return render(request, '模板文件路径', {})

  模板变量使用:{ { 模板变量名 } }
  模板for循环:
  { % for ... in ... % }
    不为空时的逻辑
  { % empty % }
    为空时的逻辑
  { % endfor %}

    模型类创建:
class 模型类名(models.Model):

    类属性名=models.字段类型(选项参数)
  # ...

模板类生成表:
1) 生成迁移文件: python manage.py makemigrations
2) 迁移生成表: python manage.py migrate

  • 10.了解Django的Admin后台管理功能【本地化，创建管理员，注册模型类，自定义模型Admin管理类】
