day04反馈

  • 1.理解并掌握状态保持的第二种方案(session)【session的读写和应用】

    session概念: 保存在服务器端的文本信息，默认保存在服务器端的django_session表中。
sesssion特点:
1) session也是以键值对进行存储的。
2) session依赖于cookie, session信息对应的唯一标识默认保存在sessionidcookie中。
3) session是有过期时间的，默认两周之后过期。

设置session: request.session['键']='值'
获取session: 变量 = request.session['键']
删除session: request.session.flush()
设置session的过期时间: request.session.set_expire('秒数')
扩展:
1) django_session表中是由INSTALLED_APPS中的'django.contrib.sessions'应用生成的。
2) session的存储需要借助于MIDDLEWARE_CLASSES中的
'django.contrib.sessions.middleware.SessionMiddleware'中间件。
3) 可以通过SESSION_ENGINE配置session存储在什么地方。

  • 2.掌握模板文件的功能及其使用。

    模板目录配置:
TEMPLATES = [
  {
      # ...
      'DIRS': [os.path.join(BASE_DIR, 'templates')],
      # ...
  }
]

模板使用过程:
1) 加载模板文件，获取模板对象
from django.template import loader
temp = loader.get_template('模板文件路径')
2）定义模板上下文，给模板文件传递数据
from django.template import RequestContext
context = RequestContext(request, {})
3) 模板渲染：替换模板变量，产生替换后的html页面
res_html = temp.render(context)

render简写函数:
from django.shortcuts import render
return render(request, '模板文件路径', {})
  • 3.了解模板文件的加载顺序。
    加载模板:
1) 先去django配置文件中配置的模板目录去找模板文件。
2) 如果找不到，会依次去INSTALLED_APPS注册的应用下的templates文件中寻找模板文件，
前提是应用中必须存在templates文件夹。

  • 4.掌握模板变量的使用及常见的模板标签和过滤器的使用。

    模板变量: 使用模板变量时，无论是使用字典的键值，对象的属性和方法，还是列表的元素，都要通过.来使用。

模板过滤器:
功能: 对模板变量进行操作。
使用方式: 模板变量|过滤器:参数

  • 5.掌握自定义模板过滤器的方法。

    自定义模板过滤器:
1) 在应用下方创建templatetags文件夹，在templatetags文件夹下创建\__init__文件。
2）在templatetags文件夹中创建py文件。
3）在创建的py文件中自定义过滤器函数。
  a）导入Library模型类
  from django.template import Library
  b）创建Library类的实例对象
  register = Library()
  c) 自定义过滤器函数并使用register对象的filter属性对其进行装饰。
  @register.filter
  def mod(num):
    return num % 2 == 0
注意: 自定义模板过滤器时，过滤器函数至少有一个参数，最多有两个参数。

自定义模板过滤器的使用:
2）使用方式和内置过滤器一致。

  • 6.掌握模板继承的语法及其使用。

    模板继承: 重用html的内容。
模板继承的语法: { % extends '父模板文件路径' % }

使用:
1）在父模板中可以预留块，子模板继承父模板之后可以重写继承的块中的内容，继承块之外的内容不能重写。
2）子模板可以在继承的块中预留新块。

预留块的语法:
{ % block 块名 % }{ % endblock 块名 % }

  • 7.掌握url反向解析的作用及其使用。

    概念: 根据url的配置项动态的生成url地址。
使用:
1) 在项目的urls文件中包含应用的urls文件。
urlpatterns = [
  url(r'^', include('应用的urls文件', namespace='值'))
]
2）在应用的urls文件中配置时指定name。
urlpatterns = [
  url(r'^index$', views.index, name='index')
]

模板文件中使用url反向解析:
无参数: { % url 'namespace:name' % }
带位置参数: { % url 'namespace:name' 参数... % }
带关键字参数: { % url 'namespace:name' 关键字参数 % }

代码中使用url反向解析:
1) 导入reverse函数
from django.core.urlresolvers import reverse
2）使用
无参数: reverse('namespace:name')
带位置参数: reverse('namespace:name', args=位置参数)
带关键字参数: reverse('namespace:name', kwargs=关键字参数)
