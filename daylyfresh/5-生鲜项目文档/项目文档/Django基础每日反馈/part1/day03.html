  • 2.理解模型管理器对象objects及自定义模型管理器类的使用场景【models.Manager】

      每个模型类默认都有一个objects(models.Manager类的实例对象)属性，
  这个objects对象叫做模型管理器对象，通过它可以操作模型类对应的数据
  表。

  自定义模型管理器类的方式:
  1) 自定定义类，继承models.Manager。
  2) 在对应的模型类中创建自定义模型管理器类的对象。
  class BookInfoManager(models.Manger):
    """图书模型管理器类"""
    # 使用场景
    # 1) 改变原有函数查询的结果集
    # 2) 封装方法：操作模型类对应的数据表

    # 管理器对象都有一个属性model，代表管理器对象所在模型类的类名

  class BookInfo(models.Model):
    """图书模型类"""
    # ...
    # 自定义模型管理器类的对象
    objects = BookInfoManager()

  • 10.理解并掌握状态保持的第一种方案(cookie)【cookie的读写和应用】

    cookie概念: 是由服务器生成，保存在浏览器端的一小段文本信息。
cookie特点:
1) cookie是以键值对进行存储的。
2) 浏览器访问网站时，会将本地保存的跟网站相关的所有cookie发送给网站的服务器。
3) cookie是基于域名安全的。
4) cookie是有过期时间的，默认关闭浏览器之后过期。

设置cookie: set_cookie(key, value, max_age='过期时间')。
读取cookie: request.COOKIES(保存浏览器发过来的cookie信息)
删除cookie. delete_cookie(key)
