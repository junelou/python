day08反馈

  • 1.了解redis数据库的特点。
    redis是一个内存型的key-value类型的非关系型数据库，支持数据的持久化，经常被用来做缓存。
  • 2.了解redis数据库的安装和配置。

  • 3.掌握redis服务器的启动和客户端连接服务器的操作。

    1）使用原有虚拟机的启动和停止
启动：sudo service redis start
停止：sudo service redis stop

2) 自己安装redis的启动和停止
启动：sudo redis-server 'redis配置文件路径'
停止：sudo kill -9 'redis服务进程的id'

3) 客户端连接服务器
redis-cli -h 'redis服务器主机ip' -p 'redis服务的端口号'
  • 4.掌握redis中的数据类型和其基本操作。
    string: 字符串，二进制安全，值最大容量为512M。
hash: 哈希，值为属性和值，属性的值只能是string。
list: 列表，值只能是string。
set: 无序集合，无顺序，值唯一并且只能是string。
zset: 有序集合，有顺序，值唯一并且只能是string。

  • 5.掌握python代码和redis数据库的交互。

    1) 安装redis包
pip install redis

2) 创建redis连接对象
from redis import StrictRedis
sr = StrictRedis(host='redis服务器主键ip', port='redis服务端口号', db='数据库编号')

3) 调用sr对象的对应方法即可实现相关操作。

  • 6.掌握django中使用redis存储session信息。

    1) 安装包
  pip install django-redis-sessions==0.5.6

2) 在settings.py文件中增加配置
SESSION_ENGINE = 'redis_sessions.session'
SESSION_REDIS_HOST = 'localhost'
SESSION_REDIS_PORT = 6379
SESSION_REDIS_DB = 2
SESSION_REDIS_PASSWORD = ''
SESSION_REDIS_PREFIX = 'session'

  • 7.了解redis中主从的搭建。

    主从redis服务的特点:
  1) 从服务会备份主服务中的数据
  2) 通过主从配置可以实现读写的分离。

从服务的配置文件中需要配置slaveof配置项:
slaveof '主redis服务ip' '主redis服务端口号'

  • 8.了解redis中集群的搭建。

    redis集群:
  一组redis服务，共同对外提供服务。

创建redis集群:
  redis-trib.rb -create --replicas 数字 redis服务ip:port ...
  注意: 自己安装的虚拟机，redis-trib.rb在/usr/local/redis/src/redis-trib.rb

redis集群的特点:
  1) 创建redis集群时，集群中的每个redis服务都是集群一个节点(node)，节点之间是有主从之
  分的，从node会备份主node中的数据。--replicas 数字：指定创建集群时一个主节点拥有几个从节点。
  2) 集群主节点会相互发送信息。
  3) 创建集群时，集群有个概念：槽(slots)，redis集群默认分为16384个slots，创建集群时会把槽
  分配到集群的每个主节点上。
  4）集群中设置键值:
    set key value
    会使用算法crc16(key)%16384 == 余数(就是键值对应该写入的槽的编号）。
  5）集群中的主节点挂掉之后，它的从节点中会有一个变成主节点。
  6) 必须要3个或以上的主节点，否则在创建集群时会失败，并且当存 活的主节点数⼩于总节点数的⼀半
  时，整个集群就⽆法提供服务了。

连接到redis集群:
  redis-cli -h 'redis节点的ip' -c -p 'redis节点的端口号'
