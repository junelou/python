day02反馈

  • 1.了解mysql配置生成mysql日志文件
    cd /etc/mysql/mysql.conf.d/
sudo vi mysqld.conf 并把68，69行注释去除
sudo service mysql restart # 重启mysql服务
cd /var/log/mysql
其中mysql.log就是mysql的日志文件，保存对mysql的操作记录
  • 2.掌握Django配置使用mysql数据库【DATABASES配置项】
    修改django项目中的settings.py配置文件
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'sz07', # 使用的mysql数据库名
        'HOST': 'localhost', # 数据库服务器主机的ip
        'PORT': 3306, # 数据库服务的端口号
        'USER': 'root', # 登录mysql数据库的用户名
        'PASSWORD': 'mysql' # 登录用户的密码
    }
}
  • 3.掌握返回应答时页面的重定向【HttpResponseRedirect或redirect】
    重定向：视图函数在处理完成之后，不给浏览器返回要显示的内容，而是让浏览器再访问另外一个url地址，这个叫做的页面重定向。
http协议方面的理解：重定向时响应报文的状态码是302，并且响应头包含Location来指定让浏览器重定向访问的url地址。
Django视图中重定向操作:
  return HttpResponseRedirect('重定向url地址')
  return redirect('重定向url地址') # 简写函数，返回值类型就是HttpResponseRedirect对象
  • 4.了解定义模型类时常见的字段属性和选项【类属性名=models.字段类型(选项)】
  • 5.掌握常用的查询相关函数(方法作用，方法参数，返回值类型)【get, all, filter, exclude, order_by】
方法名 作用 返回值 备注
get 返回满足条件的数据 返回值是模型类的对象 参数写查询条件，如果查不到数据，会发生DoesNotExist异常，如果查到多条数据，会发生MultipleObjectsReturned异常
all 返回模型类对应表格中的所有数据 返回值是QuerySet类的实例对象(查询集) 无参数
filter 返回满足条件的数据 返回值是QuerySet类的实例对象(查询集) 参数写查询条件
exclude 返回不满足条件的数据 返回值是QuerySet类的实例对象(查询集) 参数写查询条件
order_by 对查询的结果集进行排序 返回值是QuerySet类的实例对象(查询集) 参数是位置参数，写排序的字段，默认是升序，如果需要降序，需在排序字段前加-
  • 6.掌握F对象和Q对象的作用
    F对象：用于查询时条件时类属性的比较。
Q对象：用于查询时多个条件之间的逻辑关系。
使用之前需要先导入:
  from django.db.models import F,Q
  • 7.掌握aggregate聚合函数和count函数的使用
    aggregate：查询时的聚合操作，返回值是字典类型。
count：返回结果集中数据的数目。
进行聚合时要导入相应的聚合类：
  from django.db.models import Sum,Count,Avg,Max,Min
  • 8.了解查询集(QuerySet)的特性和查询集的结果集限制
    两大特性：惰性查询和缓存。
限制查询集：可以对查询集进行取下标和切片操作，切片会产生新的查询集，并且下标不能为负数。
  • 9.掌握一对多模型类之间关系的建立，多对多和一对一了解即可【ForeginKey, ManyToManyField, OneToOneFiled】
    一对多关系时在多类中定义关联属性，多对多和一对一时在任意一端类中定义关联属性。
  • 10.掌握一对多的关联查询【查询和对象关联的信息、通过模型类实现关联查询】
    通过模型类实现关联查询时，要查询哪个表中的数据，就通过对应的模型类.objects来调用对应的方法。
写关联条件时，如果模型类中有关联属性，直接写关联属性，否则需要写对应模型类的名字。
